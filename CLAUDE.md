# Project Instructions

## Verify before showing

Always verify visual/UI changes before presenting them to the user. Start the
dev server and take a preview screenshot (using the relevant viewport — desktop
AND mobile when responsive work is involved), confirm the result matches the
intent, and only then report back. Never describe a change as done without
having actually looked at the rendered result first.

## Figma MCP

Always use the local Figma Desktop MCP server for any Figma-related tasks.

- **Server:** `figma` via `http://127.0.0.1:3845/mcp`
- **Figma file:** [The-AI-Creative-Director](https://www.figma.com/design/jSbGQonaWkWeD3tCfbiNgq/The-AI-Creative-Director?m=dev)

Never use the Claude Desktop app MCP connection or a personal access token — the local Figma Desktop MCP handles authentication automatically.

If the MCP is not connected, ask the user to open Figma Desktop before continuing.
